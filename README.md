# بوت مكتبة الكتب العربية الإلكترونية عبر واتساب (الإصدار 2.0)

مشروع احترافي متكامل لبناء وتطوير بوت واتساب يعمل كمكتبة كتب رقمية شاملة. يسمح البوت للمستخدمين بالبحث عن الكتب وتنزيلها واقتراح إضافة كتب جديدة، بينما يمنح المسؤولين (الأدمن) لوحة تحكم كاملة من داخل واتساب لإدارة المكتبة ومراجعة الطلبات ورفع الكتب مباشرة إلى مستودع Hugging Face Datasets.

---

## 🚀 التقنيات المستخدمة

- **Node.js** (إصدار 18 فما فوق).
- **whatsapp-web.js** للاتصال بحساب واتساب عبر الويب بشكل مؤتمت.
- **Express.js** كخادم ويب خفيف للتأكد من حالة البوت (Health check).
- **SQLite** كقاعدة بيانات محلية لتخزين سجلات الكتب والمستخدمين والتحميلات.
- **Hugging Face Datasets** لتخزين ملفات الكتب المعتمدة بصيغة PDF وتوفير روابط تحميل مباشرة وسريعة.
- **Winston** لتسجيل وحفظ السجلات والأخطاء (Logs).

## ⚙️ متطلبات الإعداد والتشغيل

### 1. المتطلبات الأساسية
- تثبيت [Node.js](https://nodejs.org/) (الإصدار 18 أو أحدث).
- حساب على [Hugging Face](https://huggingface.co/) وإنشاء Dataset خاصة أو عامة.
- رمز وصول من Hugging Face (Access Token) مع إعطائه صلاحية **كتابة (Write)**.
- هاتف جوال مثبت عليه تطبيق واتساب لمسح رمز QR.

### 2. التثبيت والإعداد
قم بفتح نافذة الطرفية (Terminal) في مجلد المشروع، ثم اتبع الخطوات التالية:

1. **تثبيت المكتبات المطلوبة:**
   ```bash
   npm install
   ```

2. **تهيئة ملف البيئة (.env):**
   قم بنسخ ملف `.env.example` وتسميته بـ `.env` وتعديل القيم بما يناسب إعداداتك:
   ```bash
   cp .env.example .env
   ```
   تأكد من تعديل الإعدادات التالية داخل `.env`:
   - `HF_TOKEN`: رمز وصول Hugging Face الخاص بك وصلاحيته Write.
   - `HF_DATASET`: مسار الـ Dataset التي أنشأتها (مثال: `username/my-books-dataset`).
   - `ADMIN_NUMBER`: رقم المسؤول على واتساب بالصيغة الدولية متبوعاً بـ `@c.us` (مثال: `966500000000@c.us`).

3. **تشغيل المشروع:**
   ```bash
   npm start
   ```

4. **ربط الحساب:**
   عند التشغيل للمرة الأولى، سيظهر رمز QR كبير في الطرفية. افتح تطبيق واتساب على جوالك واختر **الأجهزة المرتبطة -> ربط جهاز**، ثم قم بمسح الرمز.
   سيتم حفظ الجلسة محلياً بمجلد `.wwebjs_auth` لتجنب إعادة مسح الرمز عند التشغيل مستقبلاً.

## 📱 كيفية عمل البوت (Workflows)

### 📤 1. اقتراح وإضافة كتاب جديد:
1. يقوم المستخدم بإرسال ملف الـ PDF مباشرة للبوت.
2. يتحقق البوت تلقائياً من أن الملف هو PDF حقيقي وغير تالف وحجمه لا يتجاوز الحد المسموح (50MB افتراضياً).
3. يطلب البوت من المستخدم إدخال المعلومات خطوة بخطوة (اسم الكتاب، اسم المؤلف، اختيار التصنيف من القائمة، وصف مختصر أو تخطي).
4. بعد اكتمال المدخلات، يتم حفظ الطلب بحالة `WAITING` ويتم إرسال إشعار فوري للأدمن مع إرفاق ملف الـ PDF الأصلي له.

### ⚙️ 2. معالجة وقبول الكتاب من الأدمن:
1. يرسل الأدمن رسالة: `قبول [رقم_الطلب]`.
2. يتم تغيير حالة الطلب لـ `PROCESSING`.
3. يقوم البوت بالعمليات التالية تلقائياً:
   - حساب بصمة الملف (SHA256) للتأكد من عدم وجود نسخة مكررة بالفهرس.
   - استخلاص عدد الصفحات وحجم الملف الفعلي.
   - رفع ملف الـ PDF إلى Hugging Face وتخزينه في المجلد `books/pdf/`.
   - تسجيل الكتاب بقاعدة البيانات SQLite وتحديث حالة الطلب لـ `APPROVED`.
   - إعادة بناء الفهرس المؤقت بالذاكرة فورياً.
   - إشعار المستخدم فوراً بأنه تم اعتماد كتابه ونشره للجميع.

## 💬 قائمة الأوامر المتوفرة

### 👤 أوامر المستخدمين:
- `/start` أو `القائمة`: ترحيب وعرض القائمة الرئيسية.
- `بحث`: يبدأ جلسة البحث عن كتاب (بالعنوان، المؤلف، أو الكلمات المفتاحية).
- `تصنيفات`: تصفح الكتب المتوفرة مقسمة حسب التصنيفات.
- `جديد`: جلب آخر 5 كتب تمت إضافتها للمكتبة.
- `مفضلة`: عرض قائمتك المفضلة من الكتب.
- `اضافة`: شرح وتفعيل عملية تقديم كتاب جديد.
- `احصائيات`: عرض إحصائيات عامة وسريعة للمكتبة.
- `تحميل [UUID]`: تحميل كتاب وإرساله كملف PDF مباشر.
- `مفضلة [UUID]`: إضافة كتاب إلى قائمتك المفضلة.
- `حذف_مفضلة [UUID]`: إزالة كتاب من المفضلة.

### 👑 أوامر الأدمن (المرسلة من رقم المسؤول فقط):
- `طلبات`: عرض جميع طلبات اقتراحات الكتب المعلقة (WAITING).
- `قبول [رقم_الطلب]`: الموافقة على الطلب ورفعه لـ Hugging Face وتسجيله بالمكتبة.
- `رفض [رقم_الطلب] [السبب]`: رفض الطلب وتوضيح السبب وإشعار المقترح مع تنظيف الملف المؤقت.
- `حذف [UUID أو الاسم]`: حذف كتاب من قاعدة البيانات والـ Cache.
- `تعديل [UUID] العنوان:الجديد | المؤلف:الجديد`: تعديل معلومات كتاب.
- `إحصائيات`: جلب تقرير كامل ومفصل لإحصائيات البوت والكتب والتحميلات والمستخدمين النشطين.
- `رسالة جماعية [نص الرسالة]`: بث رسالة جماعية لجميع المشتركين المسجلين في البوت.
- `نسخة احتياطية`: إرسال ملف قاعدة البيانات SQLite مباشرة للأدمن عبر واتساب للاحتفاظ بنسخة خارجية.
- `إعادة بناء الفهرس`: تحديث فهرس الذاكرة المؤقتة (Cache) للكتب فوراً.
- `تنظيف`: مسح كافة الملفات والطلبات المؤقتة التالفة.
- `مزامنة`: فحص الاتصال بمستودع Hugging Face والتأكد من صحة الصلاحيات.

## 🔒 الأمان والحماية
1. **التحقق من سلامة الملفات:** لا يكتفي البوت بالامتداد بل يفحص البايتات السحرية (Magic Bytes `%PDF`) لمنع رفع ملفات ضارة أو تنفيذية.
2. **منع التكرار:** يتم حساب SHA256 للكتب المرفوعة ومطابقتها قبل القبول لمنع تكرار نفس الكتاب بالمكتبة.
3. **محاربة السبام (Rate Limiting):** تحديد فترات انتظار بين الرسائل (ثانية واحدة افتراضياً) وفترة انتظار بين طلبات الإضافة للمستخدم الواحد (3 دقائق) لحماية الموارد.
4. **عزل التخزين:** التخزين المؤقت للملفات يجري في مجلدات معزولة يتم تنظيفها أولاً بأول.
